# Prompt 3: Auth Pack — Provider Contract & Clerk Implementation

You are a senior React Native authentication engineer. Your task is to **implement** the auth feature pack for the CLI starter, producing real templates and pack code that integrate with the existing generator.

## Context

Prompt 1 is fully implemented. Prompt 2 delivers the design system, ThemeProvider, MMKV storage, and reanimated. The auth pack (`src/packs/auth/index.ts`) is currently a stub. Your job is to fill it in.

|---|---|---|

### What the auth pack must do

1. Only run when `config.auth !== 'none'` (pack registry already gates this).
2. Populate `dependencies`, `expoInstallPackages`, and `ownedPaths` for the Clerk provider.
3. Place EJS templates under `templates/auth/` that generate into the target project.
4. Run `postApplyValidation` that checks critical files exist.

## Auth Modes

- `none`: No auth dependencies installed. No auth files generated. Route guard skips auth gating. Onboarding flows directly to app shell.
- `clerk`: Full Clerk SDK, auth provider, typed screens, OAuth support.

## Hard Requirements

### Library Stack (Auth Pack)

| Library | Purpose | Install via |
|---|---|---|
| `react-hook-form` | Performant form state for sign-in/sign-up/forgot-password screens | `expoInstallPackages` |
| `@hookform/resolvers` | Connects Zod schemas to React Hook Form | `dependencies` |
| `zod` | Schema validation for auth forms — shared between client validation and API contracts | `expoInstallPackages` |
| `@clerk/clerk-expo` | Clerk React Native SDK | `dependencies` |
| `expo-secure-store` | Secure token cache for Clerk sessions | `expoInstallPackages` |
| `expo-web-browser` | OAuth redirect handling (Google, Apple, GitHub sign-in) | `expoInstallPackages` |
| `expo-auth-session` | OAuth session management, redirect URI generation | `expoInstallPackages` |

**Why React Hook Form + Zod over Formik + Yup:**

| Criterion | React Hook Form + Zod | Formik + Yup |
|---|---|---|
| Re-renders | Minimal — uncontrolled by default, subscribes to individual fields | Every keystroke re-renders entire form |
| Bundle size | ~9 KB (RHF) + ~14 KB (Zod) | ~33 KB (Formik) + ~25 KB (Yup) |
| TypeScript | First-class inference from Zod schemas | Manual type annotations required |
| RN integration | `Controller` component wraps any RN input cleanly | Works but verbose with `handleChange`/`handleBlur` |
| Maintenance | Very active (2025-2026 releases) | Formik largely unmaintained since 2023 |
| Schema reuse | Zod schemas can be reused for API response validation | Yup schemas are client-only |

### Auth Provider Contract

Define `src/providers/auth/auth.interface.ts`:

```ts
interface AuthProvider {
  isAuthenticated: boolean
  isLoading: boolean
  user: AuthUser | null

  signIn(params: SignInParams): Promise<AuthResult>
  signUp(params: SignUpParams): Promise<AuthResult>
  signOut(): Promise<void>
  resetPassword(email: string): Promise<void>

  signInWithOAuth(provider: OAuthProvider): Promise<AuthResult>

  verifyEmailCode(params: VerifyEmailCodeParams): Promise<AuthResult>
  resendVerificationCode(params: ResendVerificationCodeParams): Promise<void>

  getToken(): Promise<string | null>
  refreshSession(): Promise<void>
}
```

Also define: `AuthUser`, `SignInParams`, `SignUpParams`, `VerifyEmailCodeParams`, `ResendVerificationCodeParams`, `AuthResult`, `OAuthProvider`, `AuthError`.

### Zod Schemas for Auth Forms

Generate `src/providers/auth/auth.schemas.ts` with:

```ts
import { z } from 'zod'

export const signInSchema = z.object({
  email: z.string().email('Please enter a valid email'),
  password: z.string().min(8, 'Password must be at least 8 characters'),
})

export const signUpSchema = z.object({
  email: z.string().email('Please enter a valid email'),
  password: z.string().min(8, 'Password must be at least 8 characters'),
  confirmPassword: z.string(),
}).refine((data) => data.password === data.confirmPassword, {
  message: 'Passwords do not match',
  path: ['confirmPassword'],
})

export const forgotPasswordSchema = z.object({
  email: z.string().email('Please enter a valid email'),
})

export const verifyEmailSchema = z.object({
  code: z.string().length(6, 'Code must be 6 digits'),
})

export type SignInFormData = z.infer<typeof signInSchema>
export type SignUpFormData = z.infer<typeof signUpSchema>
export type ForgotPasswordFormData = z.infer<typeof forgotPasswordSchema>
export type VerifyEmailFormData = z.infer<typeof verifyEmailSchema>
```

### Clerk Implementation (`clerk` mode)

1. Implement under `src/providers/auth/clerk/`.
2. Use `@clerk/clerk-expo` with `expo-secure-store` token cache.
3. Map Clerk-specific errors to typed `AuthError` codes:
   - `INVALID_CREDENTIALS`, `EMAIL_NOT_VERIFIED`, `ACCOUNT_EXISTS`, `NETWORK_ERROR`, `RATE_LIMITED`, `UNKNOWN`
4. Validate `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` at startup.
5. OAuth via `expo-web-browser` + `expo-auth-session`:
   - `useWarmUpBrowser()` hook to pre-warm browser on mount
   - Redirect URI via `AuthSession.makeRedirectUri()`

### Auth API Surface

1. `src/providers/auth/useAuth.ts` is the **only** API imported by screens. Returns the `AuthProvider` interface.
2. `src/providers/auth/index.ts` resolves implementation via config/resolver pattern (already stubbed in core).

### Auth Screens (`clerk` mode)

All screens use React Hook Form with Zod resolvers and design system tokens.

1. **`app/(auth)/sign-in.tsx`**
   - Email + password fields via `Controller` + design system `TextInput`
   - OAuth buttons (Google, Apple) via `signInWithOAuth`
   - "Forgot password?" link
   - "Don't have an account? Sign up" link
   - Loading state with disabled submit
   - Error display mapped from `AuthError` codes

2. **`app/(auth)/sign-up.tsx`**
   - Email + password + confirm password fields
   - Zod `.refine()` for password match
   - On success → redirect to verify-email
   - Same OAuth options

3. **`app/(auth)/forgot-password.tsx`**
   - Email field only
   - Success → show confirmation message
   - Rate-limit handling

4. **`app/(auth)/verify-email.tsx`**
   - 6-digit OTP input (individual digit boxes, not a single field)
   - Auto-submit when all digits filled
   - Resend cooldown timer (60s)
   - Paste support from clipboard

### Screen Pattern (all auth screens follow this)

```tsx
import { useForm, Controller } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { signInSchema, type SignInFormData } from '@/providers/auth/auth.schemas'
import { useAuth } from '@/providers/auth/useAuth'

export default function SignInScreen() {
  const { signIn } = useAuth()
  const { control, handleSubmit, formState: { errors, isSubmitting } } = useForm<SignInFormData>({
    resolver: zodResolver(signInSchema),
    defaultValues: { email: '', password: '' },
  })

  const onSubmit = async (data: SignInFormData) => {
    const result = await signIn(data)
    // handle result...
  }

  return (
    // Use design system tokens for styling
    // Controller wraps each input
    // errors.email?.message, errors.password?.message for inline validation
  )
}
```

### 5. Add-New-Provider Guide

5. Add conditional deps in `src/packs/auth/index.ts`
6. Update `src/providers/auth/index.ts` resolver switch case
7. Add provider-specific env validation
