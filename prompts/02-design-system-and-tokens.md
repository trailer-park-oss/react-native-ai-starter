# Prompt 2: UI Pack — Design System, Tokens & Library Adapters

You are a senior mobile design systems engineer. Your task is to **implement** the UI feature pack for the CLI starter, producing real templates and pack code that integrate with the existing generator.

## Context

Prompt 1 is fully implemented in `packages/create-rn-ai-starter/`. The core pack generates an Expo Router project with Zustand stores, TanStack Query client, and provider resolver stubs. The UI pack (`src/packs/ui/index.ts`) is currently a stub. Your job is to fill it in.

2. Place EJS templates under `templates/ui/` that the generator renders into the target project.
3. Run `postApplyValidation` that checks critical files exist.

### Zustand Persist with MMKV

The theme store already exists in the core pack. The UI pack must **augment** it:

1. Generate `src/lib/mmkv-storage.ts` that exports an MMKV instance and a Zustand `StateStorage` adapter.
2. The core template `src/store/theme.ts.ejs` should be updated (or the UI pack should generate a replacement) so that the Zustand store uses `persist` middleware backed by MMKV storage. This means `preset` and `colorMode` survive app restarts.

```ts
// src/lib/mmkv-storage.ts — generated by UI pack
import { MMKV } from 'react-native-mmkv'
import type { StateStorage } from 'zustand/middleware'

export const mmkvStorage = new MMKV()

export const zustandMmkvStorage: StateStorage = {
  getItem: (name) => mmkvStorage.getString(name) ?? null,
  setItem: (name, value) => mmkvStorage.set(name, value),
  removeItem: (name) => mmkvStorage.delete(name),
}
```

1. `ThemeProvider` at `src/design-system/ThemeProvider.tsx` must:
   - Read `preset` from the Zustand theme store
   - Read `colorMode` via `useColorScheme()` (with store override)
   - Resolve canonical tokens → pass to the active UI library's provider
   - Use `react-native-reanimated` for smooth animated color transitions on theme change
2. Runtime preset switch must work without app restart (Zustand + MMKV persist handles this).

2. Include iOS `shadowColor/Offset/Opacity/Radius` + Android `elevation` fields.

### 10. Architectural Decisions

| Decision | Chosen | Rationale | Rejected |
|---|---|---|---|
| Persist storage | react-native-mmkv | Synchronous reads, 30× faster than AsyncStorage, Zustand `persist` compatible | AsyncStorage (async, slower) |
| Animation engine | react-native-reanimated | UI-thread animations, worklets, shared values, Expo-compatible | Animated API (JS thread, limited) |
| Token architecture | Canonical tokens + library adapters | Swap UI library without changing design values | Library-native tokens only |
| Haptic feedback | expo-haptics | Native feel on interactions, simple API, Expo-managed | No haptics (less polished UX) |
