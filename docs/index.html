Configuration

CLI Flags

Pass flags directly or use the interactive prompt. Combine --yes to accept defaults.

Flag Values Default Description
--ui tamagui gluestack tamagui UI component library
--auth clerk none none Authentication provider
--preset radix-blue radix-green radix-purple radix-orange radix-cyan radix-red radix-blue Theme color preset
--yes - - Skip prompts, use defaults
Under the hood

Built on solid foundations

Every generated project follows a clean, scalable architecture with battle-tested libraries.

Expo SDK

Managed workflow with expo-router, expo-secure-store, and the full Expo ecosystem.

Zustand

Lightweight state management for onboarding, theme preferences, and app state.

TanStack Query

Data fetching with caching, background refetching, and optimistic updates.

React Hook Form + Zod

Type-safe form validation with schema-first design and minimal re-renders.

Design Tokens

Unified token system for colors, spacing, typography, and elevation — works across UI kits.

Provider Pattern

Abstract interfaces for auth, payments, and UI. Swap implementations without changing screens.

Get Started

Up and running in 3 steps

From zero to a running app in under a minute.

1

Scaffold your project

Run the CLI and pick your options — or pass flags to skip prompts.

npx @trailer-park-oss/create-rn-ai-starter my-app
2

Add your keys

If using Clerk auth, add your publishable key to .env.

EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_...
3

Start developing

Launch the dev server and open on your device or simulator.

cd my-app && npx expo start
